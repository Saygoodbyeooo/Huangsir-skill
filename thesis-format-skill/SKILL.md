# Skill: 江西财经大学本科毕业论文 DOCX 排版

## 触发条件

当用户要求生成、排版、格式化一份**江西财经大学本科毕业论文的 Word（.docx）文件**时触发本 Skill。

---

## 前提

- 使用 Python 3.8+ 和 `python-docx` 库（`pip install python-docx`）
- 所有格式参数定义在 `scripts/config.py`，不得在其他文件中硬编码
- 格式规范严格遵循《江西财经大学普通本科毕业论文指导手册》

---

## 执行流程

```
1. 收集用户信息：论文标题、姓名、学号、院系、专业、指导教师、日期
2. 询问用户是否已有章节内容（有 → 填入；无 → 生成占位骨架）
3. 调用 scripts/thesis_formatter.py 生成 .docx 文件
4. 提示用户：在 Word 中打开后右键目录区域 → 更新域
5. 如需检查已有文档，调用 scripts/style_checker.py
```

## 文档结构（按顺序生成，分页符分隔）

1. **封面页**（按学校模板）
2. **中文摘要**（含【关键词】）
3. **英文摘要**（含【Key words】）— 分页符另起
4. **目录**（TOC 域代码）
5. **正文**（新 section，切换页眉+阿拉伯数字页码）
6. **参考文献**
7. **附录**（可选）
8. **致谢**

---

## 对脚本的要求

1. 所有格式参数从 `scripts/config.py` 读取
2. 中英文字体必须同时设置（`font.name` + `rFonts eastAsia`）
3. 封面信息表用无边框表格，行高设为自动（AT_LEAST）
4. 目录使用 TOC 域代码，不手写假目录
5. 页码使用 PAGE 域代码
6. 摘要→目录 section：罗马数字页码，无页眉
7. 正文 section：阿拉伯数字页码 + 页眉（五号宋体居中+下框线）
8. 首行缩进必须使用 XML 的 `w:firstLineChars` 属性（单位：1/100字符），不得使用固定长度
9. 标题段落不设固定行距，使用倍数行距或自动行距
10. 表格默认三线表样式
11. 一级标题编号格式为 "1"、"2"（数字后无标点）

---

## 注意事项

- `python-docx` 无法自动生成目录内容，必须提示用户在 Word 中更新域
- 罗马数字页码需通过 section 的 `pgNumType` XML 设置
- 页眉下框线需通过段落底部边框的 XML 操作实现
- 页眉/页脚写入前必须彻底清空旧节点，防止文字覆盖
- 三线表需通过 XML 操作设置表格顶部/底部粗线 + 表头底部细线
